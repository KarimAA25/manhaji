# Manhaj Monthly Parent Report — proprietary IP

A short, monthly version of the term report. Sent on the last working day of each month. Lighter than the term report card — focuses on direction, not snapshots. Bilingual EN/AR.

The live HTML version of this template is at `demo/parent/report.html` and is what we'll show in pitch demos.

## Layout (single-page card, mobile-first)

### Cover band (dark navy gradient)
- School crest + name, "Monthly report · <Month> <Year>"
- Student name (EN), grade · section
- 4 quick stats: subject count, graded items this month, attendance %, rubric composite

### Section 1 — This month at a glance (the AI-drafted narrative)
- 100–120 words
- Three required beats: (1) overall direction, (2) one strength to celebrate, (3) one area to build with evidence
- Approver byline ("Reviewed by Ms X · 8 May 2026")

### Section 2 — Subject performance grid (3-col card layout)
- One mini-card per subject taken
- Per card: subject name · current grade · trend arrow vs last month · class context (avg + percentile)

### Section 3 — Manhaj rubric radar
- 6-axis radar (Analytical / Creative / Oral / Written / Participation / Homework)
- Side legend with per-axis numeric score
- Watch-flagged axes coloured differently (orange/red) instead of navy

### Section 4 — Per-subject rubric matrix
- Table: subject × axis = numeric score 1.0–5.0, colour-banded
- Lets a parent see *where in the subject* the issue / strength sits, not just an aggregate

### Section 5 — Improvement plan (3 cards)
- Three concrete next-month actions: numbered 1/2/3
- Each card: action title · 1-sentence rationale · link (to resource pack)
- One action minimum addresses the top watch axis

### Section 6 — University placement signal (G9+ only)
- Profile strength 0–100
- Historical placement tiers from the school's alumni history (top 3)
- Strongest-fit fields + watch list
- Bands disclaimer ("historical, not predictive") mandatory

### Section 7 — Month-over-month delta
- 4 mini-cards: rubric composite delta, attendance delta, homework on-time delta, weakest-axis delta
- Up/down arrows + colour coding

### Footer band
- "Generated by Manhaj · reviewed and approved by <teacher list> · <date>"
- WhatsApp reply CTA

## Cadence

| Step | When |
|---|---|
| AI draft generated | First working day of new month |
| Teacher review window | 3 working days |
| Head of section approval | 1 working day after teachers |
| Released to parents | 5th of the month |
| Parent ack | 7 days after release |

## What's the same as the term report card

- Rubric framework (always the same 6 axes)
- Voice/tone rules and AI guardrails
- Bilingual EN/AR parity
- University-fit logic (G9+)

## What's different from the term report card

- **No signatures** — monthly is lighter weight, reviewed not formally signed
- **No section-by-section narrative** — only the one cover narrative + per-subject in-line cards
- **Forward-leaning** — improvement plan is the central organising principle; in the term report, narrative is central and the plan is appended

## Why monthly cadence matters

- **Trend visibility** — parents see direction monthly instead of waiting 3 months for surprises
- **Course correction inside the term** — a Feb dip flagged in March is fixable; the same dip surfacing only on the June report card is not
- **Engagement compounding** — schools sending monthly reports see ~60% higher parent-portal logins vs term-only schools (industry benchmark)

## Where it lives in the schema

- `course_catalog`, `assessments`, `rubric_scores` feed the source data
- One row in a `monthly_reports` table per (student, year, month) — see V2 of the schema
- PDF + HTML rendered on demand; not pre-stored
